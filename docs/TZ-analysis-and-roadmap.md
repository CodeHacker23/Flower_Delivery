## 6. Что осталось сделать (по приоритету)

### Этап 6 (финал) — до запуска

1. **Комментарий при отмене** — курьер вводит причину отмены → сохраняем в заказ/отдельную таблицу → передаём админам в уведомлении.
2. **Включить проверку 200 м** — `app.penalty.skip-geo-check=false` (сейчас true для тестов).
3. **Проверить webhook ЮKassa** — пополнение депозита по факту оплаты.
4. **Статистика курьера** — день/неделя/выбранный период (сейчас только всего + месяц).
5. **Метки времени всех статусов** — для админа: когда каждый статус проставлен.

### Модуль админов (этап 4 / 5)

- Список магазинов, курьеров, заказов с фильтрами.
- Назначение/снятие заказа с курьера.
- Карта с геолокацией курьеров на линии.
- Разбор «звоночков», назначение штрафов.
- Задержка возврата комиссии (см. courier-flow-analysis-and-gaps.md, §13).

### Модуль депозитов (этап 5)

- UI настройки % комиссии в админке (поле есть, UI — нет).
- Полная интеграция ЮKassa (webhook, тесты).
