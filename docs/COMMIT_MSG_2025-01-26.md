## Body (подробное описание для GitHub)

### Реализовано по ТЗ

**Курьерский модуль:**
- **Пагинация «Доступные заказы»** — 10 заказов на страницу, кнопки «← Назад» / «Дальше →», редактирование сообщения (EditMessageText). Номера при выборе — локальные на странице (1–10).
- **Защита от сговора (штрафы):**
  - Штраф 500₽ при совпадении обоих гео (магазин + получатель в радиусе 200 м) — CourierPenaltyService.
  - При отсутствии гео (доставили без бота и отменили) — комиссию не возвращаем (вариант A).
  - Проверка 200 м при подтверждении «В магазине» / «Вручил» — реализована (getReferenceCoordinatesForPending, 3 попытки, вызов админа), но **закомментирована** для тестов. TODO: раскомментировать после полевых тестов.
- **Убрано автопополнение 1000₽** при активации курьера через `/k` — баланс только через ручное пополнение или ЮKassa.

**Магазин (создание заказа):**
- **Порядок полей по ТЗ:** адрес доставки → цена → дата → имя → телефон → комментарий.
- **Валидация цены** по тарифной сетке: при ручном вводе минимум = тариф для расстояния (app.tariffs.X). Если цена ниже — показ тарифной сетки и повторный запрос.
- **Кнопка «Отменить заказ»** на шаге с ценой — callback `order_creation_cancel`, метод `cancelOrderCreation()`.

**SQL-скрипты:**
- `scripts/deposit_topup.sql` — пополнение баланса курьера по telegram_id. Простой SQL (UPDATE + INSERT с подзапросом), без DO-блока.
- `scripts/reset_courier.sql` — сброс профиля курьера с учётом FK: order_status_geo_snapshots → order_stops → orders → courier_transactions → couriers → users.

**Документация:**
- `docs/COURIER_MODULE_STATUS.md` — актуальный статус курьерского модуля.
- `docs/SHOP_ORDER_CREATION.md` — порядок создания заказа и валидация цены.
