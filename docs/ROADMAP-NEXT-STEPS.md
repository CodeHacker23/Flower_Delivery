## 4. Штрафы 500₽ — доработать

**Текущее состояние:**
- Штраф 1000₽ — автоматически при ≥3 отменах за 30 дней.
- «Звоночек» админу — когда курьер был в магазине, но не у получателя по гео.
- Штраф 500₽ — в конфиге есть (`penaltyAmount500`), но **не применяется автоматически**.

**Варианты для штрафа 500₽:**

| Вариант | Описание |
|---------|----------|
| A | Автоматически при «звоночке» (был в магазине, не у получателя) — сразу списывать 500₽. |
| B | Оставить на усмотрение админа — «звоночек» только уведомляет, админ решает и назначает штраф вручную. |
| C | Комбинированный: при первом «звоночке» — только уведомление; при повторном за N дней — автоматический штраф 500₽. |

**Нужно решить:** какой вариант использовать.

## 5. Перерасчёт цены при отмене точки курьером

**Сейчас:** Курьер может отменить только весь заказ целиком.

**Нужно:** Возможность отменить одну точку в мультизаказе (например, 2-й адрес).

**Логика:**
1. Курьер выбирает «Отменить точку N» (а не весь заказ).
2. Точка удаляется из `order_stops`.
3. Вызывается `recalcMultiStopDelivery` — пересчёт маршрута и цены по оставшимся точкам.
4. Обновляется `order.delivery_price`.
5. Комиссия пересчитывается по новой сумме (если заказ продолжается).

**Ограничение:** Если остаётся 1 точка — заказ становится обычным (не мультиадресным). Нужно продумать переход.

## 6. Адрес доставки первым шагом у магазина

**Сейчас:** Дата → Имя → Телефон → Адрес → Цена → …

**Нужно:** Адрес → Цена → [Продолжить / Отменить] → Имя → Телефон → …

**Сценарий:**
1. Шаг 1: магазин вводит адрес доставки.
2. Геокодирование → показываем цену (и км).
3. Кнопки: «✅ Продолжить оформление» | «❌ Отменить».
4. «Отменить» — ничего не создаём в БД, возврат в меню.
5. «Продолжить» — дата, имя, телефон и т.д.

**Плюсы:** Магазин сразу видит цену и может отказаться до ввода остальных данных.

## 7. Минимальная цена по тарифной сетке

**Сейчас:**
- При ручном вводе цены — проверка `>= 300₽` (жёстко).
- Для доп. точки — минимум 300₽.

**Нужно:** Минимум = тариф для данного расстояния.

**Пример:** 7 км → тариф 500₽ → минимум 500₽. 3 км → тариф 300₽ → минимум 300₽.

**Реализация:**
- В `DeliveryPriceService`: `getMinPriceForDistance(double distanceKm)` — возвращает тариф по расстоянию.
- При ручном вводе цены — валидация против `getMinPriceForDistance(distance)`, а не против фиксированных 300₽.
- Для доп. точки — аналогично: минимум = тариф для расстояния от предыдущей точки (но не меньше 300₽ для очень близких точек).

## Приоритет

1. Адрес первым + отмена на шаге (магазин) — быстрая доработка.
2. Минимальная цена по сетке — быстрая доработка.
3. Отмена одной точки курьером + перерасчёт — средняя сложность.
4. Штраф 500₽ — после выбора варианта.
5. Связки заказов — большая задача.
6. ЮKassa — проверка.
